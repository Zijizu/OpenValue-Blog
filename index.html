Building serverless Java
applications using
Micronaut and OpenFaaS

When working on microservices in the Java ecosystem, especially using Spring (Boot), you will notice the long start-up time that application will have, let alone the high memory consumption they will have. The overhead of each single microservice will eventually take its costs on the system. Looking at a framework such as Micronaut could help out to reduce this overhead without loosing any developer's productivity. Not only "classic" applications can be build using Micronaut, but also serverless applications, functions, can be build and deployed on a cloud environment or on Kubernetes using OpenFaaS.
